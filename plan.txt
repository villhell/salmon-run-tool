# Salmon Run NW 記録ツール 要件定義書

## 概要

SplatNet 3のデータを活用して、サーモンランNWのスケジュールと
プレイ記録・統計を表示するGitHub Pagesサイト。

---

## フェーズ構成

### フェーズ1（MVP）：スケジュール表示 ✅ 実装済み

実装済みファイル構成:
```
salmon-run-tool/
├── index.html                          # スケジュールページ
├── css/style.css                       # ダークテーマ（オレンジ・深緑）
├── js/schedule.js                      # データ取得・表示ロジック
├── data/schedules.json                 # キャッシュデータ（GitHub Actionsが自動更新）
└── .github/workflows/update-schedules.yml  # 毎時データ更新
```

実装内容:
- splatoon3.ink/data/schedules.json をキャッシュ取得（fallback: 直接fetch）
- splatoon3.ink/data/locale/ja-JP.json からステージ名・武器名・オカシラ名を日本語化
- 通常バイト / ビッグラン / バイトチームコンテスト を色分けバッジで識別
- 現在シフトのカウントダウン（1秒更新）
- 武器4種を画像グリッドで表示（黒背景、ホバーで拡大）
- オカシラ予告表示（ロケールから取得）
- GitHub Pages で静的配信（ビルド不要・Vanilla JS）
- ヘッダー・ファビコンに splatoon3.ink の coop.svg アイコンを使用
- GitHub Actions で毎時スケジュールデータを自動更新
- 公開URL: https://villhell.github.io/salmon-run-tool/

オカシラ日本語名（ロケールファイルから取得）:
- Cohozuna → ヨコヅナ
- Horrorboros → タツ
- Megalodontia → ジョー
- Triumvirate → オカシラ連合

将来的にID入力での複数人対応を見据えた設計にしておく

### フェーズ2（将来）：プレイ記録の蓄積 ⚠️ 保留中

**現状の問題：**
Nintendo NSO アプリ v3.0.1（2025年5月）以降、f トークン生成 API が機能しなくなり、
s3s・nxapi を含む全自動取得ツールが認証エラーで使用不可。

調査済みの代替手段：
- s3s → f トークン生成失敗（imink API が "Something went wrong" を返す）
- nxapi → znca-api が 401 Unauthorized（要クライアント認証）
- splatnet3-token-util → Android エミュレータが必要（Android Studio 不要の軽量版あり）
- iPhone + mitmproxy → トークンが約2時間で失効するため自動化困難
- stat.ink 経由 → 上流の Nintendo 認証が同じく破綻しているため解決にならない

**再開条件：** s3s issue #198 が解決されたとき
https://github.com/frozenpandaman/s3s/issues/198

## ページ構成

### 1. スケジュールページ

データソース: splatoon3.ink/data/schedules.json（認証不要・無料）

表示内容:
- 現在のシフト（残り時間カウントダウン付き）
- 今後公開されているすべての未来シフト
- 各シフトごとにステージ・武器4種（画像）・開始〜終了時間
- オカシラ予告表示（ヨコヅナ / タツ / ジョー / オカシラ連合）
- Big Run / Eggstra Work の識別表示
- ※イベント種別（霧・ラッシュ等）はシフト前には不明のため表示不要

---

### 2. バイト記録ページ（フェーズ2以降）

データソース: s3s または s3si.ts で取得したJSON（直近50件）
※ DBが整い次第実装

表示内容（1件ごと）:

  シフト情報
  - プレイ日時・ステージ・使用武器（Wave別）
  - クリア/失敗（何Waveまでクリアか）
  - ハザードレベル

  自分の成績
  - 金イクラ納品数・赤イクラ獲得数
  - スペシャル種類・使用回数
  - 救助数・被救助数
  - オオモノ撃破数（種類別合計）

  Wave別情報
  - 各Waveの潮位・イベント種別
  - ノルマ・納品数
  - 金イクラ出現数（goldenPopCount）

  チームメンバー情報
  - 他3人の金イクラ納品数・撃破数

  オカシラ
  - 出現の有無・撃破できたか
  - 獲得スケール（銅/銀/金）

---

### 3. 統計ページ（フェーズ2以降）

集計軸: 全体 / シフト期間別 / ステージ別 / 武器別

主な統計項目:
  - クリア率（全体 / Wave別）
  - 平均金イクラ納品数
  - 平均オオモノ撃破数（種類別）
  - オカシラ撃破率
  - 武器別の平均納品数・撃破数
  - ステージ別のクリア率・納品数
  - ハザードレベル別のクリア率

シフト期間内統計（スケジュールとの紐付け）:
  - 同じシフト期間中のプレイを集計
  - 武器編成ごとの成績比較
  - ステージごとの自分の平均納品数

  ※ スケジュールの startTime と記録の startTime を照合して
    シフト単位の集計を実現する

## データ管理

| 種類             | 保存場所              | 更新頻度                    |
|------------------|-----------------------|-----------------------------|
| スケジュール     | data/schedules.json   | GitHub Actions で自動更新   |
| バイト記録       | data/results.json     | フェーズ2以降               |

- 記録は追記型にして過去データを失わないようにする（50件制限を回避）
- スケジュールIDと記録の startTime を照合してシフト単位の集計を実現

---

## データソース

| データ                 | 取得元                          | 認証   |
|------------------------|---------------------------------|--------|
| スケジュール           | splatoon3.ink API               | 不要   |
| 個人バイト記録         | SplatNet 3（s3s / s3si.ts経由） | 必要   |

### SplatNet 3 から取得できる主なデータ

Wave別:
- イベント種別（霧・ラッシュ・グリル等）
- 潮位（満潮/干潮/通常）
- 金イクラのノルマ・納品数
- 金イクラ出現数（goldenPopCount）

個人成績:
- 使用武器（各Waveで支給されたもの）
- スペシャルウェポン種類・使用回数
- 金イクラ・赤イクラの獲得数
- オオモノ撃破数（種類別、シフト合計）
- 救助回数・被救助回数

チーム:
- メンバー3人分の同上データ（名前・バイネーム含む）

キングサーモン:
- 出現有無・撃破可否
- 獲得スケール（銅/銀/金）

取得不可なデータ:
- Wave別・オオモノ種類別の撃破数（シフト合計のみ）
- Wave別・オオモノ種類別の出現数（APIに露出なし）

---

## 制約・注意事項

- SplatNet 3 は直近50件しか保存しない仕様のため、
  定期的な取得・蓄積が必須（フェーズ2のDB導入まで要検討）
- SplatNet 3 の内部APIを使用するため、
  任天堂のAPI仕様変更により動作しなくなる可能性がある
- スケジュールのイベント種別（霧・ラッシュ等）は
  シフト開始前には取得不可
